Introduction

The CSS Transition property is a simple way to animation to our website

Animation on the web increases user experience and the most basic way to animate something is with Transition.

"Transitions enable you to define transition between two states of an element." -https://developer.mozilla.org/en-US/docs/Web/CSS/transition

When the state of an element changes through things like hover, focus or something else which can change its CSS Property. Transition make the change transition from one state to another instead of instant change.

Here are some of the transition properties we are going to look at.

  • transition-duration
  • transition-property
  • transition-timing-function
  • transition-delay
  • transition

Transition-duration

It takes in a number followed by ms(milliseconds) or s(seconds). In General it is suggested to use ms, because while CSS can accept ms and s, JavaScript only accepts ms. So even if you are currently not using JavaScript, you will probably going to eventually get into it. It is handy to be using ms as a standard through everything you are doing.

Here is a sample:

.div{ width: 300px; height: 300px; background: #3DCCC7; transition-duration: 500ms; }

.div:hover{ background: #FF6B6B; }

Short durations are key for simple animations , but for complex animations one can set as required.

Transition-property

This is what is being transitioned, things like width, background, transform.... etc.

Sample:

.div{ width: 300px; height: 300px; background: #3DCCC7; transition-duration: 1000ms; transition-property: background, transform; }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

One can also do the same thing by setting it to "all".

.div{ width: 300px; height: 300px; background: #3DCCC7; transition-duration: 1000ms; transition-property: all; }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

Though this is an easy and fast way to do it, it is not encouraged. Transitions are CPU intensive. There are some that the GPU can rely on but not many. Most of the time it is the CPU that's doing it and this can cause some really junky animations. When using "all", it tries to animate everything (eg. width , height), which might slow things down. It might be working well on a super high end computer or gadget, but not on a budgeted ones.

The comma seperation style helps it to focus on the specific properties targeted making it easy on the CPU therefore better performance. That is why this is encouraged

Transition-timing-function

This is a fun one with a long wierd name. It's a wierd one especially if one is not familiar with some of the things like "ease-in", the "bezier curves". This creates a bouncy movement effect on the animation with the animation duration.

Here are the most common ones:

  • Linear
  • Ease
  • Ease-in
  • Ease- out
  • Ease-in-out

The defult is "ease".

.div{ width: 300px; height: 300px; background: #3DCCC7; transition-duration: 1000ms; transition-property: background, transform; transition-timing-function: ease-in; }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

There are actually a few more options than these ones. If you want a faster way to work than guessing what you really want to do, the "Dev tools" are fantastic ways to visualize whats going on and offer us more option to use. One can get the "cubic-bezier" of which you can tweak then copy the desired one to your code.

.div{ width: 300px; height: 300px; background: #3DCCC7; transition-duration: 1000ms; transition-property: background, transform; transition-timing-function: cubic-bezier(1,.11,.22,1.41); }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

Transition-delay

This defines how long we should wait for the transition to start happening.

Sample:

.div{ width: 300px; height: 300px; background: #3DCCC7; transition-duration: 1000ms; transition-property: background, transform; transition-timing-function: cubic-bezier(1,.11,.22,1.41); transition-delay: 2s; }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

Transition

Instead of writing out all these properties seperately, you can write them in one "transition" property. Just like there is margin, which takes values for margin- top, margin-left...etc

What is important is order you put them in.

.div{ width: 300px; height: 300px; background: #3DCCC7; //transition-duration: 250ms; //transition-property: background, transform; //transition-timing-function: cubic-bezier(1,.11,.22,1.41); // transition-delay: 2s; transition: background 250ms ease-in, transform 500ms ease-out; }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

If you take the duration "250ms" and set it as the first and add a second number, the second will take the role of transition-delay.

.div{ width: 300px; height: 300px; background: #3DCCC7; //transition-duration: 250ms; //transition-property: background, transform; //transition-timing-function: cubic-bezier(1,.11,.22,1.41); // transition-delay: 2s; transition: background 250ms ease-in, 500ms transform ease-out 1s; }

.div:hover{ background: #FF6B6B; transform: rotate(45deg); }

Conclusion

On performance, if you can limit the animations to transforms and opacity. In the code examples we are transitioning our background. In general we do want to avoid this, though most do transition background for things like button hover. For transforms and opacities, the browser uses our graphic cards for these animations so it helps them run smoothly. For others if there is no GPU acceleration we quickly run into junky animations especially if we use the transition "all".

It is fun to animate things like box-shadows and borders and a whole bunch of stuff... all at the same time, but be aware the browser is re-paintning all this and it has a hit on performance

Hope this has been a nice introduction on how to handle CSS transition, and some other thing you might do with it.
